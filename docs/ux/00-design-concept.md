## 设计理念：流体科技 (Fluid Technology)

### 核心概念
将**液态流动的有机感**与**精密的数字科技**相融合。压缩包的解压过程本质是"释放"和"流动"——文件从封闭容器中流淌而出。设计应该让用户感受到这种**数据解放的愉悦感**。

### 视觉隐喻
```
压缩包  = 流体容器
解压    = 液体倾倒
文件    = 水滴/粒子
进度    = 流动的河流
```

---

## 三个设计主张

### 1. 有机流动，非机械冰冷
**反对**: 传统解压软件的方格、直角、静态列表  
**主张**: 圆润曲线、渐变过渡、呼吸式动画

像水流过鹅卵石，而非齿轮转动。

### 2. 轻盈透明，非厚重实体
**反对**: 厚重的面板、明确的边界、强烈的对比  
**主张**: 毛玻璃效果、柔和阴影、透气的留白

像晨雾，而非混凝土墙。

### 3. 主动引导，非被动等待
**反对**: 用户不知道下一步该做什么，静态界面  
**主张**: 动效暗示操作、磁力吸附、自然的节奏

像溪流引导落叶，而非路标指示。

## 色彩策略：水色光谱

### 主色：青色渐变 (Cyan Gradient)
```
为什么选青色？
- 科技感：RGB光谱中的冷色调
- 信任感：介于蓝（理性）与绿（生机）之间
- 流动感：联想水流、液晶、全息投影

主渐变：
从 #06b6d4 (青色500)
到 #0ea5e9 (天蓝500)

光感变体：
加白光 → #67e8f9 (高光)
加深影 → #0e7490 (阴影)
```

### 辅助色：极光色盘
```
紫色 #a78bfa - 创意/特殊文件
粉色 #f472b6 - 图片/媒体文件
橙色 #fb923c - 压缩档案/重要
绿色 #34d399 - 完成状态
```

### 中性色：深海渐变
```
浅色模式 (水面之上):
  纯白 → 浅灰 → 中灰
  #ffffff → #f8fafc → #e2e8f0

深色模式 (深海之下):
  深蓝黑 → 板岩灰 → 钢灰
  #0f172a → #1e293b → #334155

理念：深色不是"黑"，而是"深海的蓝"
```

## 视觉风格：新玻璃态 (Neo-Glassmorphism)

### 核心特征
```
1. 毛玻璃背景
   backdrop-filter: blur(20px)
   background: rgba(255,255,255,0.7)

2. 柔和边界
   border: 1px solid rgba(255,255,255,0.3)
   无强烈黑色描边

3. 多层深度
   用模糊度区分层级，而非阴影大小

4. 光晕效果
   卡片顶部有微弱白色渐变（模拟光照）
```

### 与传统玻璃态的差异
```
传统 Glassmorphism:
  - 过度使用模糊（视觉疲劳）
  - 缺乏层次（所有都是玻璃）
  
新玻璃态:
  - 克制使用：仅核心卡片用玻璃
  - 混合材质：玻璃 + 实体 + 渐变
  - 功能优先：可读性 > 炫技
```

## 动效哲学：流体力学

### 三大原则

#### 1. 惯性与阻尼
所有动画都有"起步加速"和"减速停止"，不用线性匀速。

```javascript
// 弹簧物理参数
spring: {
  damping: 20,    // 阻尼（越大越快停止）
  stiffness: 300  // 刚度（越大越快运动）
}
```

#### 2. 连续性
元素消失时要有去向，出现时要有来源。

```
错误: opacity 0 → 1 (凭空出现)
正确: 从父容器边缘滑入 + 淡入
```

#### 3. 呼应性
用户操作立即有微反馈（100ms内），主动画可延后。

```
点击按钮:
  0ms   → 按钮缩小 scale(0.95)
  100ms → 涟漪扩散动画
  300ms → 执行实际操作
```

## 交互微动效库

### 按钮交互
```
Hover:
  - 轻微上浮 (translateY: -2px)
  - 阴影加深
  - 背景渐变偏移

Press:
  - 缩小至 scale(0.97)
  - 内阴影出现
  - 200ms 内弹回

涟漪效果:
  - 从点击点扩散半透明圆
  - 800ms 淡出消失
```

### 卡片交互
```
Hover:
  - 轻微放大 scale(1.02)
  - 边框发光 (box-shadow glow)
  - 内容轻微上浮

长按:
  - 背景脉冲呼吸 (opacity 0.8 ↔ 1)
  - 暗示可拖拽

拖拽中:
  - 放大 scale(1.05)
  - 强阴影 (跟随鼠标)
  - 磁力吸附线提示
```

### 列表动画
```
进入:
  - 错落出现 (stagger delay: 50ms)
  - 从下往上滑入 + 淡入
  - 轻微过冲 (overshoot)

排序:
  - 流体布局动画
  - 位置互换时有缓动曲线
```

## 布局原则：呼吸式留白

### 空间密度
```
紧凑区 (功能密集):
  间距: 8px-16px
  示例: 工具栏、文件列表

舒适区 (核心交互):
  间距: 24px-32px
  示例: 上传区、预览区

呼吸区 (视觉焦点):
  间距: 48px-64px+
  示例: 空状态、引导页
```

### 黄金比例应用
```
主内容区 : 侧边栏 = 1.618 : 1
卡片宽度 : 卡片高度 ≈ 1.5 : 1
```

## 信息架构：三层渐进

### Layer 1: 主操作层
```
用户 80% 时间在这里
- 拖拽上传区（视觉焦点）
- 文件列表（核心内容）
- 快捷操作栏（常用按钮）

设计要求: 极简、大按钮、清晰引导
```

### Layer 2: 辅助功能层
```
用户 15% 时间触达
- 搜索过滤
- 视图切换（网格/列表）
- 设置面板

设计要求: 收起默认，需要时展开
```

### Layer 3: 高级选项层
```
用户 5% 时间探索
- 批量操作
- 高级设置
- 键盘快捷键

设计要求: 渐进披露，不干扰主流程
```

## 情感化设计点

### 空状态
```
传统: "暂无文件" + 无聊图标
我们: 动态液滴动画 + "拖入压缩包，看它们流淌而出"
```

### 加载状态
```
传统: 转圈菊花
我们: 液体填充动画（像水倒入容器）
```

### 成功状态
```
传统: 绿色勾勾
我们: 粒子爆炸 + 轻微震动 + 音效（可选）
```

### 错误状态
```
传统: 红色叉叉 + 错误代码
我们: 柔和晃动 + 友好文案 + 解决建议
```

## 响应式策略

### 断点
```
sm:  640px   - 手机竖屏
md:  768px   - 手机横屏/小平板
lg:  1024px  - 平板/小笔记本
xl:  1280px  - 笔记本
2xl: 1536px  - 台式机/大屏
```

### 适配原则
```
<768px:  单列布局，底部操作栏
768-1024px: 混合布局，侧边栏可收起
>1024px: 完整布局，多列展示
```

## 可访问性承诺

### 对比度
```
正文文字: 最低 4.5:1
大标题: 最低 3:1
所有交互元素: 符合 WCAG AA 级
```

### 键盘导航
```
Tab: 按逻辑顺序遍历所有交互元素
Space: 激活按钮
Enter: 确认操作
Esc: 关闭弹窗
```

### 屏幕阅读器
```
所有图标有 aria-label
动态内容有 aria-live
复杂组件有完整 ARIA 属性
```
